# Components overview

Every Readsmith page can use the built-in component library in MDX, no imports required. This page is the map; each gallery page documents one component in depth and renders it live.

## Callouts

<Info>
Five severities share one anatomy, plus `Check` for confirmations. This paragraph sits in an `Info`. See [Callouts](/components/callouts).
</Info>

## Structure

| Component | Use for | Gallery |
| --- | --- | --- |
| `<Accordion>`, `<AccordionGroup>` | Optional detail the reader expands on demand | [Accordions](/components/accordions) |
| `<Expandable>` | Lightweight nested disclosure inside prose or a field list | [Expandable](/components/expandable) |
| `<Banner>` | A prominent top-of-page announcement, optionally dismissible | [Banner](/components/banner) |
| `<Tabs>`, `<Tab>` | One choice among alternatives, with page-wide sync | [Tabs](/components/tabs) |
| `<Steps>`, `<Step>` | Numbered procedures | [Steps](/components/steps) |
| `<Card>`, `<CardGroup>` | Linked navigation tiles | [Cards](/components/cards) |
| `<Frame>` | Media with a caption and a CLS-free declared ratio | [Frames](/components/frames) |

## Code

| Component | Use for | Gallery |
| --- | --- | --- |
| `<CodeGroup>` | The same snippet in several languages, tabbed | [Code groups](/components/code-groups) |
| ` ```mermaid ` fence | Diagrams rendered from text, themed and zoomable | [Diagrams](/components/diagrams) |

Fenced code blocks need no component: they highlight at build time with dual themes, titles, and line emphasis. See [Code blocks](/authoring/code-blocks); the diagram fence is authoring syntax too, at [Authoring, Diagrams](/authoring/diagrams).
